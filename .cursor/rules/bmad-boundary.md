# BMAD Boundary

BMAD Method is the primary delivery workflow for planning and implementation. Cursor advisory agents are review gates, not a replacement for BMAD.

## Use BMAD for delivery

Use BMAD skills/agents for:

- Discovery and product brief
- PRFAQ
- PRD creation and validation
- UX specification
- Architecture and ADRs
- Epics and stories
- Sprint planning/status
- Story implementation
- Code review
- Correct-course workflow
- Retrospective

## Use Cursor advisory agents for review

Use `.cursor/agents` only when explicitly requested or at a defined quality gate:

- After Product Brief, before PRD final
- After PRD, before Architecture
- After Architecture, before Sprint 1 implementation
- Before MVP release

## Hard boundaries

- Do not auto-call council during BMAD workflows.
- Do not auto-call all advisory agents for normal questions.
- Do not let advisory agents override BMAD phase order.
- Do not let AI agents move artifacts to the next phase without Solo PM approval.
- Do not implement out-of-MVP suggestions generated by advisory agents unless Solo PM explicitly promotes them into scope.

## Conflict handling

When BMAD output and advisory feedback conflict:

1. Summarize the conflict.
2. Identify the decision owner: Solo PM.
3. Present 2-3 options with trade-offs.
4. Wait for Solo PM decision before modifying artifacts or implementation.
